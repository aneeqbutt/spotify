Project: Spotify Automation Platform (SpotifyBot)   Category: Delivery/New Feature

Goal Description

Title: Build React Dashboard + Task Form + Run Now Feature (May 26 - Jun 1)

Problem Statement:

The backend was fully operational with JWT auth, WebSocket command routing, and an SQLite task store,
but the platform had no user interface. Developers and operators had no way to schedule tasks, select
target devices, dispatch commands, or monitor live execution progress without calling raw REST API
endpoints manually. A frontend dashboard was required to make the automation platform usable without
any API tooling.

Solution Approach:

* Set up React 18 + Vite project with Axios configured to communicate with the FastAPI backend.
* Build a device selector dropdown that fetches all registered devices and auto-selects the first
  online device on page load.
* Implement a task creation form with an action type picker (SEARCH_AND_PLAY, LIKE_CURRENT_TRACK,
  FOLLOW_ARTIST, SKIP_TRACK) and dynamic parameter inputs per action.
* Add a "Run Now" button that dispatches an immediate command to the selected device via the backend
  WebSocket pipeline.
* Display real-time task progress (STEP_STARTED, STEP_OK, STEP_FAILED, COMMAND_DONE events) using
  an SSE stream connected to the backend broadcaster.
* Style and manually smoke-test the full end-to-end flow from the dashboard to the physical device.

Acceptance Criteria:

Functional behavior: The dashboard shows all registered devices with their online/offline status.
The task form accepts an action type and its required parameters. The Run Now button triggers an
immediate command routed to the selected device. Live step progress updates are visible in the UI
as the device executes each step, ending with COMMAND_DONE.

Performance metrics: The dashboard loads and renders the device list within 2 seconds of page open.
Device status changes (connect/disconnect) are reflected in the UI within 1 second via SSE. Run Now
dispatches the command and the first STEP_STARTED event appears in the UI within 2 seconds.

Tests written: Manual smoke test covering the full Run Now flow end-to-end with a physical device
connected over the local network. Verified STEP_OK events stream correctly and COMMAND_DONE closes
the run.

Reviewed & deployed: Code committed and running locally at http://localhost:5173 against the FastAPI
backend on port 8000. Validated on Samsung Galaxy S21 FE with the APK installed.
